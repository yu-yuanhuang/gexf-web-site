# 社會網絡圖互動展示（GitHub Pages）
##GEXF做成可互動網站
繪製社會網絡圖、格式GEXF
目標：網絡圖格式能在網站上展示、調整展示條件
可以控制要顯示哪些點／哪些邊
獨立網頁github佈署
原始數據保護

網頁端通常用「過濾（filter）」達成
**依節點屬性**類別（community／group）、國家、機構、年份、角色等（GEXF需有node attributes）
**依網絡指標**degree、weighted degree、betweenness、pagerank、k-core 等
**依邊權重／關係類型**只顯示 weight ≥ 門檻的連線；或只顯示某種 relation
**子圖模式**Mouse Hover以「選到的節點」為中心，只顯示其1-hop/2-hop鄰居（ego network）
**搜尋驅動**使用者輸入關鍵字，只顯示命中節點＋其鄰居

##控制方式
預設載入全部資料，但「先隱藏一部分」再依UI切換顯示

- 左側控制面板（繁中）
  - 節點清單：可逐一【顯示／不顯示】（含搜尋、全選／全不選／反選）
  - 四項指標門檻（已繁中化）
    - 邊權重門檻（weight）
    - 邊數門檻（degree）
    - 加權邊數門檻（weighted degree）
    - 中介中心性門檻（betweenness）
  - 【顯示孤立節點】開關
- 右側互動大圖
  - 滑鼠滾輪放大／縮小、拖曳平移
  - 右上角：放大＋、縮小－、重置視角
  - Hover節點顯示詳細資料（含計算的degree／weighted degree與讀取到的betweenness）
效能優化：算圖在佈署前完成，增加前端處理效能

1.網絡圖的「邊」線條過細，【顯示控制】在顯示孤立節點旁，多一個邊寬度選項拉桿，預設邊寬度請用這一版的3倍粗 2.控制介面多一個【顯示：邊】/【不顯示：邊】開關，可以控制圖譜要不要看邊 3.節點標籤在不改顯示比例時，至少展示70%的節點標籤，當使用者放大時再依序展示80%、90%、100%節點標籤 4.當用戶變更背景顏色時，右上角的【放大】【縮小】【重置】按鈕配色要用【背景的對比色】當按鈕底色
2.標籤顯示比例（縮放分級 70/80/90/100%）
- 初始比例（未縮放）至少顯示 **70%** 節點標籤。
- 以「初始視角」為基準，使用者放大後依序提升到 **80% → 90% → 100%**。
- 觸發是「跨過門檻」才更新，避免縮放過程中反覆重算造成卡頓。
3.背景變更時右上角【放大／縮小／重置】按鈕底色改用背景對比色
- 當背景色改變（含自訂色）時，會自動計算對比色（黑／白）並套用到右上角按鈕底色，同時調整文字顏色確保可讀性。
4.Hover 節點時：所有連結節點顯示節點標題
- 滑鼠移到任一節點時，該節點的**所有鄰接節點**會強制顯示其「節點標題（label）」；非鄰接節點會淡化並隱藏標籤，讓關聯關係更清楚。
